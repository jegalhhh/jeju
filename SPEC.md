## 1. 프로젝트 개요

### 서비스명
- **정식명** —`느린 소식`
- **부제**: 제주ver

### 컨셉
제주도는 오랫동안 섬이었다. 육지로 소식을 전하려면 파도가 잠잠해지길 기다려 배에 실어야 했고, 언제 닿을지 알 수 없었지만 그래도 보냈다. 제주에는 소식을 전하는 새조차 없었다. 까치는 제주 땅을 밟지 않았고, 소식은 오직 바람과 파도에 실려야만 닿을 수 있었다.

이 서비스는 그 감성을 현대적으로 재해석한다. 제주 여행 중 남긴 소원과 편지가, 여행이 끝난 뒤 사용자의 일상으로 배달된다. 기다림과 설렘, 그리고 뒤늦게 닿는 감동.

### 타겟
- 제주 게스트하우스 여행객
- 개인 / 그룹 여행자 모두
- 앱 설치 없이 즉시 참여 가능 (번호 인증만)

### 핵심 가치
- **시간차 감동**: 여행 중 입력 → 여행 후 수신
- **제주 스토리텔링**: 성산일출봉 소원, 배편 서신이라는 제주 고유의 맥락
- **낮은 진입장벽**: 로그인 없이 링크 하나로 참여

## 3. 핵심 기능

### 기능 1. 성산일출봉 소원 (봉이)

#### 배경
성산일출봉은 매일 아침 떠오르는 해를 보며 간절한 소원을 비는 제주 대표 일출 명소다. 이 서비스는 그 소원을 디지털로 담아, 약속된 날 사용자에게 다시 전달한다.

#### 사용자 플로우
```
① 메인에서 '성산일출봉 소원 빌기' 선택
② 이름 + 전화번호 입력 → SMS 인증번호 발송 → 인증 완료
③ 제주에서 찍은 사진 업로드
④ 소원 텍스트 입력
⑤ 소원 받을 날짜 선택 (귀국일 또는 원하는 날)
⑥ 제출
⑦ 봉이(Claude Haiku)가 사진 + 소원을 읽고 짧은 편지 즉시 자동 생성
⑧ /wish/[uuid] 페이지 즉시 생성 (편지 포함, SMS 발송 전이라도 URL 접근 가능)
⑨ 지정일 당일 Cron이 SMS로 링크 자동 발송
⑩ 사용자가 링크 열면 사진 + 소원 + 봉이 편지 확인
```

#### 발송 웹페이지 (`/wish/[uuid]`)
- 봉이 캐릭터 + 제주 감성 헤더
- 사용자가 업로드한 사진
- 사용자가 적은 소원
- 봉이가 쓴 편지
- 하단 카피: *"이 소원이 이루어지길 바랍니다"*

#### SMS 발송
- 관리자 승인 불필요, Cron이 지정일 당일 자동 발송
- SMS 본문: 링크만 포함 (MMS 아님)

---

### 기능 2. 배편 서신 (바람이)

#### 배경
과거 제주도는 중요한 서신을 배로 육지에 보냈다. 파도와 바람이 허락해야 소식이 닿았다. 이 기능은 그 기다림의 감성을 그룹 편지 형식으로 재현한다.

#### 방 입장 구조
별도 URL이나 코드 없이, 웹 목록에서 방을 클릭해 입장한다.

```
메인 → '배편 서신 편지 쓰기'
     → 전체 방 목록 UI (시작 전 방 우선 노출)
     → 원하는 방 클릭 → 입장
```

- 모든 방(시작 전/시작 후)이 목록에 노출됨
- **시작 전 방**: 클릭 가능, 목록 상단에 우선 노출
- **시작된 방**: 클릭 불가(비활성 표시), 목록 하단에 밀려 표시
- 방 시작 후에는 신규 입장 불가

#### 관리자 플로우
```
① /admin 접속 → 비밀번호 입력
② 방 만들기 (방 이름 + 발송일 설정)
③ 방이 웹 목록에 자동 노출
④ 참여자 입장 현황 실시간 확인
⑤ 다 모이면 "방 시작" → 순환 매칭 자동 생성
⑥ 편지 작성 완료 목록 확인
⑦ 각 편지 내용 + 사진 미리보기 (발송 페이지 링크 포함)
⑧ 승인 or 반려
   └ 반려 시 작성자에게 재작성 요청 SMS 자동 발송
⑨ 발송일 당일 Cron이 승인된 편지 SMS 자동 발송
⑩ 발송 상태 확인 (대기중 / 발송완료 / 실패)
```

#### 참여자 플로우
```
① 메인 → '배편 서신 편지 쓰기'
② 방 목록에서 원하는 방 클릭
③ 이름 + 전화번호 입력 → SMS 인증 → 참여 완료
④ 관리자가 방 시작 전까지 대기
⑤ 방 시작 후 매칭 상대 이름 공개
⑥ 매칭 상대에게 편지 + 사진 작성 후 제출
⑦ 발송일 당일 SMS 링크 수신
⑧ 링크 열면 전용 웹페이지에서 편지 + 사진 확인
```

#### 매칭 방식
- **순환 매칭**: 참여자 목록을 랜덤 셔플 후 순서대로 연결
- 예: [민수, 지수, 현수] → 민수→지수, 지수→현수, 현수→민수
- 홀수/짝수 인원 무관하게 항상 성립 (`members[i] → members[(i+1) % n]`)
- 관리자도 참여자로 등록 가능 (관리자 본인 편지는 자동 승인)

#### 발송 웹페이지 (`/letter/[uuid]`)
- 바람이 캐릭터 + *"배편으로 도착한 편지입니다"* 헤더
- 보내는 사람 이름
- 첨부 사진
- 편지 내용
- 하단 제주 감성 마무리 카피

#### 웹페이지 생성 시점
- 편지 제출 즉시 `/letter/[uuid]` 페이지 생성
- SMS 발송 전이라도 URL 접근 가능 상태
- 관리자는 /admin에서 링크 클릭해 미리보기 가능
- 사용자는 SMS가 와야 링크를 알게 되는 구조

## 4. 관리자 페이지 (`/admin`)

### 접근
- 비밀번호 입력으로 접근 (별도 계정 없음)
- 비밀번호는 환경변수로 관리

### 성산일출봉 소원 관리
| 항목 | 내용 |
|------|------|
| 소원 목록 | 이름 / 소원 요약 / 발송 예정일 / 발송 상태 |
| 발송 상태 | 대기중 / 발송완료 / 실패 |
| 미리보기 | /wish/[uuid] 링크 클릭으로 발송 페이지 확인 |

### 배편 서신 방 관리
| 항목 | 내용 |
|------|------|
| 방 목록 | 방 이름 / 참여자 수 / 발송일 / 상태 |
| 방 관리 | 방 만들기 / 시작하기 |
| 참여자 | 참여자 목록 및 매칭 결과 확인 |
| 편지 검토 | 내용 + 사진 미리보기 / 승인 / 반려 |
| 발송 관리 | /letter/[uuid] 링크 미리보기 / 발송 상태 확인 (참여자별) |

## 5. 공통 사항

### 인증
- 로그인 없음
- 전화번호 입력 시 SMS 인증번호로 본인 확인

### SMS 발송 정책
| 종류 | 발송 시점 | 승인 필요 |
|------|----------|----------|
| 번호 인증 | 즉시 자동 | 불필요 |
| 성산일출봉 소원 링크 | 지정일 Cron | 불필요 |
| 배편 서신 편지 링크 | 발송일 Cron | **관리자 승인 필요** |
| 배편 서신 반려 알림 | 반려 즉시 자동 | 불필요 |

### 웹페이지 생성 정책
- 사용자 제출 즉시 고유 UUID 페이지 생성
- SMS 발송 전에도 URL 접근 가능
- 관리자만 링크를 미리 알고 있음 (사용자는 SMS 수신 후 접근)

### 어뷰징 방지
- 배편 서신 편지는 관리자 수동 검토 후 승인
- 반려 시 작성자에게 재작성 요청 SMS 자동 발송
- 성산일출봉 소원은 개인 수신이므로 별도 검토 없음

## 7. 캐릭터 설정

### 봉이 (성산일출봉 소원)
- 정체: 성산일출봉을 지키는 빛의 정령
- 역할: 매일 아침 일출과 함께 소원을 듣고 편지를 씀
- 말투: 따뜻하고 시적, 짧고 간결
- 기술: Claude Haiku API (사진 + 소원 텍스트 입력 → 편지 생성)

### 바람이 (배편 서신)
- 정체: 제주 바다를 건너 소식을 전하는 바람의 정령
- 역할: 편지를 배에 실어 약속된 날 전달
- 말투: 잔잔하고 기다림의 감성
- 기술: 캐릭터 UI (AI 생성 없음, 전달자 역할만)
