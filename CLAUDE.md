# CLAUDE.md — 느린 소식 (봉이와 바람이)

## 필독 파일

작업 시작 전 반드시 아래 두 파일을 먼저 읽을 것.

- `SPEC.md` — 서비스 기획, 기능 정의, 플로우, 기술 스택, 디렉토리 구조, 환경변수
- `DESIGN.md` — 디자인 시스템, 컬러, 타이포그래피, 컴포넌트 규칙

---

## 코딩 규칙

- Claude Haiku API, SOLAPI SMS 발송은 반드시 서버 사이드에서만 수행 (API 키 노출 금지)
- Supabase 민감한 쿼리는 반드시 RLS 적용
- 이미지는 Supabase Storage에 저장, DB에는 URL만 저장
- 환경변수는 절대 클라이언트 코드에 노출하지 않는다

---

## SPEC.md 동기화 규칙

코드 구현 중 `SPEC.md`와 다른 방향으로 결정된 사항이 생기면 반드시 아래 절차를 따른다.

1. 변경된 내용을 코드에 반영한다
2. `SPEC.md`의 해당 항목을 즉시 수정한다
3. `SPEC.md` 상단 버전 번호를 올린다 (예: v2.0 → v2.1)

> 코드와 SPEC.md는 항상 일치 상태를 유지한다.
